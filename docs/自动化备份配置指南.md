# 自动化备份配置指南

本文档说明如何配置自动化备份，确保代码持续同步到远程仓库。

---

## 📋 备份策略

### 策略1：定时任务（推荐）

使用操作系统的定时任务功能，定期执行备份脚本。

#### Windows任务计划程序

1. **打开任务计划程序**
   - 按 `Win + R`，输入 `taskschd.msc`
   - 或搜索"任务计划程序"

2. **创建基本任务**
   - 右侧操作面板点击"创建基本任务"
   - 名称：`CryptoMonitor-AutoBackup`
   - 描述：`定期自动备份代码到GitHub`

3. **配置触发器**
   - 选择"每天"或"每小时"（根据需求）
   - 设置开始时间

4. **配置操作**
   - 选择"启动程序"
   - 程序/脚本：`C:\Program Files\Git\bin\bash.exe`
   - 参数：`-c "cd D:/工作文件 && bash scripts/auto_backup.sh"`
   - 起始于：`D:\工作文件`

5. **完成配置**
   - 勾选"打开此任务的属性对话框"
   - 在属性中，勾选"使用最高权限运行"

#### Linux/Mac cron

编辑crontab：
```bash
crontab -e
```

添加以下行（每小时备份一次）：
```bash
0 * * * * cd /path/to/工作文件 && bash scripts/auto_backup.sh
```

### 策略2：Git Hooks

使用Git的pre-commit hook，在每次提交前自动执行备份。

创建 `.git/hooks/pre-commit`：
```bash
#!/bin/bash
# 自动推送
git push origin main
```

---

### 策略3：WorkBuddy自动化任务

使用WorkBuddy的自动化功能，定期执行备份。

配置示例：
- **名称**：`代码自动备份`
- **提示词**：`执行自动备份脚本：cd D:/工作文件 && bash scripts/auto_backup.sh`
- **计划**：每天 23:00
- **工作目录**：`D:\工作文件`

## 🔄 多重备份配置

### 备份到多个远程仓库

1. **添加多个远程仓库**
   ```bash
   git remote add backup1 https://gitee.com/你的用户名/crypto-monitor-system.git
   git remote add backup2 https://gitlab.com/你的用户名/crypto-monitor-system.git
   ```

2. **修改备份脚本，推送到所有远程仓库**
   ```bash
   # 推送到所有远程仓库
   git push origin main
   git push backup1 main
   git push backup2 main
   ```

## 📝 提交信息规范

本项目遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范。

### 提交格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type类型
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关
- `chore(auto-backup)`: 自动备份

### 示例
```
feat(monitor): 添加ETH大额委托数据监控

- 集成CoinGlass API获取ETH大额委托
- 优化推送格式，防止企微截断
- 更新22维度分析框架

Closes #123
```

## 🛡️ 安全注意事项

1. **API密钥不要提交到仓库**
   - 已添加到 `.gitignore`
   - 使用环境变量或配置文件（不提交）

2. **Token不要存储在代码中**
   - 使用 `.git-credentials` 或环境变量
   - 定期更换Token

3. **定期检查提交历史**
   - 确保没有敏感信息泄露
   - 使用 `git log --patch` 检查

## 📞 故障排查

### 推送失败

**问题**：`remote: Write access to repository not granted`

**解决方案**：
1. 检查Token权限（Fine-grained PAT需要配置仓库访问）
2. 检查远程仓库URL是否正确
3. 重新生成Token

### SSL连接失败

**问题**：`schannel: failed to receive handshake, SSL/TLS connection failed`

**解决方案**：
1. 配置Git凭据存储：`git config --global credential.helper store`
2. 创建 `~/.git-credentials` 文件
3. 或使用SSH密钥认证
